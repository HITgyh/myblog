# 自动维护脚本日期说明

## 当前日期获取方式

### ✅ 已实现：使用文件创建时间（birthtime）

自动维护脚本现在使用**文件的实际创建时间**作为文章的 `date` 字段：

```javascript
date: new Date(birthTime).toISOString().split('T')[0]
```

### 🔍 工作原理

1. 脚本在扫描文件时，获取每个 Markdown 文件的文件系统元数据
2. 使用 `stat.birthtime`（文件创建时间）或 `stat.mtime`（最后修改时间）作为备选
3. 将时间转换为 `YYYY-MM-DD` 格式

### ⚠️ 重要说明

由于我们之前将文件从 `C:\Users\Guo\Desktop\codeStudyNote` 复制到 `myblog/server/posts/` 目录，**复制操作会创建新的文件**，因此：

- **当前日期**：所有文件的创建时间都是今天（2026-04-01），这是复制到 posts 目录的时间
- **原始文件的创建时间**：保存在 `C:\Users\Guo\Desktop\codeStudyNote` 目录中的原始文件保留了更早的创建时间

### 🛠️ 如何保留原始日期

如果你希望文章日期使用**原始文件的创建时间**，而不是复制后的时间，有两个方案：

#### 方案 1：直接从原始目录扫描（推荐）
修改脚本，从原始目录 `C:\Users\Guo\Desktop\codeStudyNote` 扫描文件，而不是从 `posts` 目录。

#### 方案 2：使用文件修改时间
如果你的文件在本地有定期的修改，文件修改时间（mtime）可能更能反映文件的活跃时间。
